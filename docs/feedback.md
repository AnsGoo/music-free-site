# 反馈与参与

感谢你愿意帮助改进 MusicFree。请通过 [GitHub Issues](https://github.com/ansgoo/music-free-site/issues) 提交反馈，便于跟踪与回复。

## 提交前请先确认

1. **搜索已有 Issue**：避免重复提交相同问题或建议。
2. **选择正确模板**（在 [新建 Issue](https://github.com/ansgoo/music-free-site/issues/new/choose) 页面）：
   - **Bug 报告**：MusicFree 服务端、管理后台、插件宿主等行为问题。
   - **文档问题**：本站说明错误、链接失效、示例过时等。
   - **功能特性**：新功能或体验改进建议。
3. **尽量提供可复现信息**：版本号、操作步骤、日志或截图能显著加快处理速度。

## 功能建议

**[提交功能建议 →](https://github.com/ansgoo/music-free-site/issues/new?template=feature_request.md)**

建议在描述中说明：

- 使用场景与要解决的问题
- 期望的功能行为
- 可选：参考产品、截图或相关链接

## Bug 报告

**[提交 Bug →](https://github.com/ansgoo/music-free-site/issues/new?template=bug_report.md)**

请尽量填写：

| 信息 | 说明 |
| ---- | ---- |
| MusicFree 版本 | 镜像标签、发布版本或 commit |
| 环境 | 部署方式（Docker / 裸机）、操作系统、相关插件 |
| 复现步骤 | 从操作到出现问题的完整路径 |
| 实际 vs 期望 | 当前结果与你认为正确的结果 |

## 文档问题

**[提交文档问题 →](https://github.com/ansgoo/music-free-site/issues/new?template=doc_report.md)**

请说明相关页面路径、问题描述与期望的正确内容。
